## รายละเอียดโดยย่อ
Project นี้เป็น App คลังสินค้า โดยเราจะมีการอัปเดตจำนวนทั้งขาเข้าและขาออกเพื่อทราบจำนวนของสินค้าในระบบ โดยมีการระบุรายละเอียดของสินค้าไว้

## คุณลักษณะและขอบเขต
- สามารถเพิ่มสินค้าใหม่ (ชื่อสินค้า, ราคา, จำนวนคงเหลือ)
- สามารถแก้ไขข้อมูลสินค้า
- สามารถลบสินค้าออกจากระบบ
- แสดงรายการสินค้าทั้งหมดในรูปแบบตาราง
- มีระบบค้นหาสินค้าตามชื่อหรือรหัสสินค้า
- มีระบบบันทึกข้อมูล (Save) ลงไฟล์
- มีระบบโหลดข้อมูล (Load) จากไฟล์กลับมาใช้งาน
- มีอย่างน้อย 2 หน้า (เช่น หน้าเมนูหลักที่แสดงรายการสินค้า และหน้าจัดการสินค้า(เพิ่มสินค้า/ แก้ไขสินค้า))

### คุณลักษณะขั้นต่ำ
- สามารถเพิ่มสินค้าใหม่เข้าสู่ระบบ
- ชื่อสินค้า
- ราคาต่อหน่วย
- หน้าเมนูหลัก
- หน้าจัดการสินค้า
- จำนวนเริ่มต้น
- สามารถอัปเดตจำนวนสินค้าในคลัง
- ให้เพิ่มหมวดหมู่สินค้าเองได้
- เพิ่มจำนวนสินค้า (รับเข้า)
- ลดจำนวนสินค้า (ขายออก)
- ตรวจสอบไม่ให้จำนวนสินค้าติดลบ
- คำนวณยอดรวมมูลค่าสินค้ารับเข้า
- คำนวณยอดรวมมูลค่าสินค้าขายออก
- สามารถบันทึกข้อมูล (Save) และโหลดข้อมูล (Load) ได้
- มีอย่างน้อย 2 หน้าจอ
- เพิ่มช่อง search ใน Dashboard
- เพิ่ม delete product/modify product
- เปลี่ยน database เป็น mongo db หรื่ออื่น ๆ
- เปลี่ยนID ให้Genขึ้นมาเอง หรือ ไม่ต้องมี หรือ ให้Match กับ barcode =>mongo DB genให้

### คุณลักษณะเพิ่มเติม
- ระบบแจ้งเตือนเมื่อสินค้าคงเหลือน้อย
- ระบบคำนวณมูลค่าสินค้าคงคลังทั้งหมด
- ระบบเพิ่ม/ลดจำนวนสินค้า (รับเข้า-ขายออก)
- ระบบจัดหมวดหมู่สินค้า
- ระบบสรุปรายงานสินค้าเป็นไฟล์ PDF

## แผนการดำเนินโครงงาน
ร่างแผนการดำเนินโครงงานในระยะเวลา 2 เดือน (8 สัปดาห์) โดยระบุผลลัพธ์ที่คาดหวังว่าจะทำได้เสร็จในแต่ละระยะ เช่น

| **ระยะ**        | **ช่วงเวลา**   | **ผลลัพธ์ที่คาดหวัง**                                                                            |
| --------------- | -------------- | ------------------------------------------------------------------------------------------------ |
| เริ่มต้นโครงงาน | สัปดาห์ที่ 1-2 | - ส่งข้อเสนอโครงงาน<br>- สร้าง GitHub repository<br>- สร้างโครงงานด้วย Maven และเขียนโค้ดตั้งต้น |
| สร้างต้นแบบ     | สัปดาห์ที่ 3-4 | - โปรแกรมทำงานได้ในเบื้องต้น<br>- มีฟีเจอร์หลัก (core feature) ที่ใช้งานได้                  |
| ทดลองใช้งาน     | สัปดาห์ที่ 5-6 | - ฟีเจอร์ส่วนใหญ่พร้อมใช้งาน<br>- โปรแกรมสามารถใช้งานได้ แต่อาจต้องปรับแก้หรือเพิ่มเติม          |
| พร้อมส่งมอบงาน  | สัปดาห์ที่ 7-8 | - โปรแกรม/เกมมีฟีเจอร์ครบถ้วนตามที่วางแผน<br>- เอกสารประกอบสมบูรณ์ พร้อมสำหรับการนำเสนอ          |
**หมายเหตุ**

- ฟีเจอร์หลัก (core feature) หมายถึงฟีเจอร์สำคัญที่ทำให้โปรแกรม/เกมบรรลุวัตถุประสงค์หลัก
- MVP (Minimum Viable Product) คือเป้าหมายขั้นต่ำที่สามารถส่งมอบได้ อย่างไรก็ตาม ฟีเจอร์หลักอาจเป็น subset ของ MVP ได้ในบางกรณี

## ความท้าทายและความเสี่ยง
1. **ความท้าทายด้านเทคนิค**:
   - ยังไม่ชำนาญภาษา Java และ JavaFX
   - การจัดการไฟล์ข้อมูล (File I/O)
   - **แนวทางแก้ไข**:
   - ศึกษาเพิ่มเติมจากเอกสารออนไลน์และวิดีโอสอน
   - ทดลองเขียนโปรแกรมย่อยก่อนนำมารวมในโปรเจกต์จริง
2. **ความท้าทายด้านการการออกแบบระบบ**:
   - การออกแบบโครงสร้างคลาสให้เหมาะสม
   - การออกแบบ UI ให้ใช้งานง่าย
   -**แนวทางแก้ไข**:
   - วางแผนโครงสร้างโปรแกรมก่อนเริ่มเขียนโค้ด
   - ออกแบบหน้าจอกระดาษก่อนพัฒนา
3. **ความเสี่ยงด้านเวลา**:
   - เวลาในการเรียนรู้เครื่องมือใหม่
   - อาจเกิดปัญหา Bug ระหว่างพัฒนา
   - **แนวทางแก้ไข**:
   - แบ่งงานให้ชัดเจน
   - ทำงานเป็นช่วง ๆ ไม่รอใกล้กำหนดส่ง
   - ทดสอบระบบทุกครั้งหลังเพิ่มฟีเจอร์
สร้างระบบที่สามารถจัดการสินค้าได้โดยไม่ทำให้ผู้ใช้งานเกิดความสับสนในตัวแอพ
